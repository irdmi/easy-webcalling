# Serverless Video Call — видеозвонки без сервера

Бесплатное веб-приложение для видеозвонков, работающее полностью в браузере. Не нужен ни собственный сервер, ни регистрация — просто откройте сайт и звоните. Хостится на GitHub Pages.

## Возможности

- **Прямое P2P-соединение** — видео и звук передаются напрямую между браузерами (WebRTC), без центрального сервера.
- **Один клик для звонка** — вы создаёте комнату, копируете короткую ссылку и отправляете другу; он переходит по ней и сразу подключается.
- **Опциональный пароль** — можно включить защиту: в ссылке будет только публичный префикс, а секретная часть (пароль) показывается создателю, и её нужно передать отдельно.
- **Сквозное шифрование** — аудио и видео шифруются протоколом DTLS‑SRTP; никто посторонний не сможет подслушать разговор.
- **Прозрачность соединения** — боковая панель показывает ваш локальный IP, список удалённых ICE‑кандидатов (с IP и типом), а также текущее состояние ICE и соединения.
- **Управление микрофоном и камерой** — во время звонка можно отключить или включить микрофон и камеру.
- **Адаптивный дизайн** — работает на компьютерах и мобильных устройствах.
- **Бюджет: 0 рублей** — используется только бесплатная инфраструктура (сигнальный сервер PeerJS, GitHub Pages).

## Как это работает

1. **Создание комнаты** — при нажатии «Create Call» генерируется случайный идентификатор (например, `abc123`).  
2. **Отправка ссылки** — идентификатор (или его публичный префикс при включённом пароле) помещается в хеш URL (`#abc123`). Вы копируете ссылку и отправляете собеседнику.  
3. **Подключение** — друг открывает ссылку, страница читает хеш и «входит» в комнату.  
4. **Сигнализация** — оба браузера соединяются с бесплатным сигнальным сервером PeerJS (`0.peerjs.com`). Этот сервер только передаёт технические данные (SDP и ICE‑кандидаты), но **не имеет доступа** к вашему видео и звуку.  
5. **Прямой видеозвонок** — как только браузеры нашли друг друга, между ними устанавливается прямое зашифрованное соединение, и медиапоток идёт в обход сервера.  
6. **Разговор** — вы видите и слышите друг друга напрямую.

Если включена **защита паролем**:
- Полный ID комнаты делится на **короткий префикс** (6 символов в ссылке) и **секретную часть** (пароль).
- При подключении нужно ввести пароль. Без него сигнальный сервер не сможет связать двух участников, и звонок не состоится.

## Быстрый старт

### Разместить у себя (бесплатно)

1. **Скопируйте репозиторий** к себе на GitHub (форк).
2. Убедитесь, что файл называется `index.html`.
3. Перейдите в настройки репозитория → **Pages**.
   - В разделе *Source* выберите **Deploy from a branch**.
   - Укажите ветку `main` и папку `/ (root)`.
   - Нажмите **Save**.
4. Через пару минут сайт станет доступен по адресу `https://ваш-логин.github.io/имя-репозитория/`.
5. Откройте эту ссылку, разрешите доступ к камере и микрофону, и звоните.

Никаких других файлов, сборок или серверов не требуется.

### Запуск локально

Поскольку приложение использует ES‑модули и WebRTC, открывать файл напрямую через `file://` нельзя. Запустите локальный сервер:

```bash
# В папке с проектом выполните:
python3 -m http.server 8000
# или
npx serve .
```

Затем откройте в браузере `http://localhost:8000`.

## Как пользоваться

1. **Создатель звонка** (тот, кто начинает):
   - Зайдите на сайт.
   - При желании отметьте галочку **«Protect with password»**.
   - Нажмите **«Create Call»**.
   - Появится ссылка. Нажмите **📋 Copy Link**, чтобы скопировать её.
   - Отправьте ссылку другу.
   - Если был включён пароль, передайте пароль отдельно (например, продиктуйте по телефону).

2. **Присоединяющийся** (друг):
   - Откройте полученную ссылку.
   - Если потребуется, введите пароль и нажмите **«Join»**.
   - Разрешите браузеру использовать камеру и микрофон.
   - Звонок начнётся автоматически — вы увидите оба видео.

Во время разговора можно:
- Отключить микрофон кнопкой **🎤 Mute Mic**.
- Отключить камеру кнопкой **📷 Mute Cam**.

В боковой панели отображается ваш IP, все ICE‑кандидаты собеседника (с IP и типом) и текущее состояние соединения — так вы всегда видите, как именно установлена связь.

## Безопасность и конфиденциальность

| Аспект | Что видно |
|--------|-----------|
| **Содержимое разговора** | Полностью зашифровано (DTLS‑SRTP). Ни сигнальный сервер, ни кто‑либо в сети не может расшифровать. |
| **Сервер сигнализации (`0.peerjs.com`)** | Видит ваш IP, ID комнаты и время подключения. **Не имеет доступа** к медиапотоку. |
| **ID комнаты** | При включённом пароле в ссылке только короткий префикс — полный ID угадать невозможно. Без пароля полный ID в ссылке, поэтому храните ссылку в тайне. |
| **Атака «человек посередине»** | Теоретически владелец сигнального сервера мог бы подменить сигнальные сообщения и перехватить звонок. На практике для обычных звонков это крайне маловероятно. Для максимальной защиты нужна полностью децентрализованная сигнализация (см. планы на будущее). |
| **IP‑адреса** | Видны сигнальному серверу и вашему собеседнику. Это особенность любого прямого P2P‑соединения. |

> **Важно:** Не публикуйте ссылку в открытом доступе, если не используете пароль.
